Entscheidungs-Log

01
Wo leben generierte Assets?
Pollinations Media Storage (S3-kompatibel). Metadaten in IndexedDB.
02
Wie werden Pruna-Responses gehandelt?
Pruna gibt Binary zurück → sofort zu Pollinations Media hochladen → stabile URL. Kein direkter Blob-Storage im Client.
03
Warum Pruna als separater Provider statt Pollinations-Modell?
Pruna hat eigenes Auth (apikey-Header), async Submit+Poll statt direkter Response, und eigene Modell-IDs — kein gemeinsamer Codepfad möglich.
04
Code-Mode — warum kein eigener UI-Tab?
Code-Mode ist ein Response-Mode-Flag (CODE_REASONING_SYSTEM_PROMPT), kein separates Werkzeug. Sauberere Trennung.
05
Doku-Form für dieses Projekt?
Eine interaktive overview.html statt vieler Markdown-Dateien. Gleiche Struktur wie heyhireset/overview.html — cross-Verlinkung geplant.
